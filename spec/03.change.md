Task 6 — TopAppBar icons: fix grid_view and view_sidebar (idle + active states)
File: loom/ide/src/components/TopAppBar.tsx
Current bug: Both icons are non-functional — neither responds correctly in idle state nor in active state.
Required behavior, split by icon:
grid_view icon

Clicking it opens the Agent Panel modal (the agent-selector/agent-list modal referenced in Task 5/Task 2 context).
Must work identically whether the app is in idle state (no prompt sent yet) or active state (prompt/session already running). Currently it's likely wired to only fire post-first-prompt — remove that dependency.
Clicking the icon again (or clicking outside, per the outside-click rule below) should close it.

view_sidebar icon

Clicking it opens the file explorer overlay showing the selected project's file tree.
Must work in both idle and active states — this is the core bug: right now the tree only populates after the first prompt is sent. Trigger the file-tree fetch/render directly off "project selected," not off "first prompt sent."
Disabled state: view_sidebar icon itself should be disabled (not clickable, visually greyed/disabled styling) whenever no project is selected, regardless of idle/active state. Enable the moment a project is selected.

Shared overlay behavior (applies to whichever panel is open — Agent Panel or file explorer)

No layout shift: Render as a floating overlay (position: fixed or absolute, high z-index) on top of existing UI. Do not let it push/resize/reflow sibling panels — it must not be a flex/grid layout sibling.
Outside-click to close: Clicking anywhere outside the open panel's bounding box closes it. Use an outside-click ref/listener (or the codebase's existing outside-click hook if one exists — check before writing a new one).
Toggle button still works: Clicking the same icon again while its panel is open must still close it (don't let outside-click handling break the manual toggle).
Mutually exclusive (confirm before building): If both panels use the same overlay slot, opening one while the other is open should close the first — flag this assumption to the user/dev if the codebase suggests otherwise, don't assume silently.

Acceptance criteria:

grid_view opens Agent Panel modal in both idle and active state.
view_sidebar opens file tree in both idle and active state, populated immediately (no dependency on a prompt being sent).
view_sidebar is disabled when no project is selected; enabled the moment one is.
Opening either panel causes zero layout shift elsewhere.
Clicking outside either open panel collapses it.
Clicking the triggering icon again still manually toggles it closed/open.
